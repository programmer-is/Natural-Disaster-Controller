## ✨ داستان تولد این سامانه

تصور کن زلزله آمده. همه جا شلوغ است، خطوط تلفن قطع شده‌اند و کسی نمی‌داند کدام منطقه خطر بیشتری دارد.  
درست در همین لحظه، **یک نقشهٔ آنلاین** روی مانیتور مرکز فرماندهی روشن است که نشان می‌دهد:

- کدام استان قرمز است (خطرناک)،  
- چه هشدارهایی فعال است،  
- و مردم عادی چه گزارش‌هایی فرستاده‌اند.

**Natural Disaster Controller** دقیقاً همان سامانه‌ای است که این قدرت را به تو می‌دهد.  
نسخه‌ای که الان در دست توست، طوری طراحی شده که **هر کسی**، حتی بدون هیچ پیش‌زمینهٔ فنی، بتواند آن را روی کامپیوتر شخصی خودش راه بیندازد و تمرین مدیریت بحران کند.

## ⚙️ جعبه ابزار جادویی (فناوری‌ها)

برای اینکه سامانه کار کند، چند ابزار به‌صورت هماهنگ کنار هم قرار گرفته‌اند. نگران نباش، لازم نیست تک‌تکشان را بشناسی—فقط بدان که تیم قدرتمندی پشت صحنه مشغول کارند:

- 🐘 **PHP** → مغز متفکر سامانه  
- 🗄️ **MySQL** → حافظهٔ امن و چابک  
- 🎨 **HTML5 / CSS3** → زیبایی و چیدمان صفحات  
- ⚡ **JavaScript** → زنده کردن بخش‌های پویا  
- 📱 **Bootstrap** → سازگاری با موبایل، تبلت و دسکتاپ  
- 🔄 **AJAX** → به‌روزرسانی اطلاعات بدون رفرش شدن صفحه  
- 🌐 **WampServer** → آشپزخانه‌ای که همه را کنار هم می‌آورد و روشن می‌کند

## 🚀 راه‌اندازی قدم‌به‌قدم – بدون ذره‌ای کدنویسی

> اگر تا حالا اسم «کد» را هم نشنیده‌ای، همین حالا لبخند بزن و ادامه بده.  
> درست مانند دنبال کردن دستور پخت یک کیک ساده، فقط کافی است مرحله‌ها را پشت سر بگذاری.

### 📋 پیش‌نیازها (چیزهایی که قبل از شروع لازم داری)

- یک کامپیوتر با ویندوز (۷، ۸، ۱۰ یا ۱۱)  
- حدود ۲۰۰ مگابایت فضای خالی  
- مرورگر (Chrome، Firefox یا Edge)  
- فایل‌های پروژه که همین الان دست توست  
- یک فنجان چای یا قهوه ☕ (اختیاری ولی پیشنهاد می‌شود!)

---

### ۱. نصب WampServer

**WampServer** برنامه‌ای رایگان است که PHP و MySQL را یکجا روی ویندوز روشن می‌کند.

1. به سایت رسمی برو: [www.wampserver.com](https://www.wampserver.com)  
2. نسخهٔ مناسب ویندوزت (۳۲ یا ۶۴ بیتی) را دانلود کن.  
3. فایل دانلودی را اجرا کن و مراحل نصب را با گزینه‌های پیش‌فرض ادامه بده.  
4. بعد از نصب، WampServer را اجرا کن.  
5. حالا در گوشهٔ پایین صفحه، کنار ساعت ویندوز، یک آیکون کوچک **W** می‌بینی.  
   - 🟢 اگر سبز باشد = عالیه! همه چیز آماده است.  
   - 🟠🔴 اگر نارنجی یا قرمز بود، به بخش [رفع اشکال](#-اگر-چیزی-درست-کار-نکرد-رفع-اشکال) برو.

---

### ۲. کپی پروژه در محل درست

WampServer همه پروژه‌ها را از یک پوشهٔ مخصوص می‌خواند:

1. پوشهٔ پروژه‌ای که از ما گرفتی (پر از فایل و پوشه‌های مختلف) را **دقیقاً همانطور که هست** کپی کن.  
2. به مسیر زیر برو:
   ```
   C:\wamp64\www
   ```
3. پوشه را داخل `www` بچسبان.  
   ⚠️ اگر نام پوشه فاصله دارد، برای راحتی کار نامش را تغییر بده و به جای فاصله، زیرخط `_` بگذار.  
   مثال:  
   `C:\wamp64\www\Natural_Disaster_Controller`

---

### ۳. ساختن پایگاه داده

پایگاه داده مثل یک کمد بزرگ اطلاعات است که سامانه از آن تغذیه می‌کند. ساختنش فقط چند کلیک وقت می‌برد.

1. مرورگر را باز کن و در نوار آدرس تایپ کن:
   ```
   http://localhost/phpmyadmin
   ```
   (حتماً WampServer سبز باشد، وگرنه این صفحه بالا نمی‌آید.)  
2. در صفحهٔ باز شده، روی دکمهٔ **New** (یا «جدید») در ستون سمت چپ کلیک کن.  
3. یک نام برای پایگاه داده وارد کن:  
   ```
   natural_disaster_db
   ```
4. از منوی کشویی کنار آن، گزینهٔ `utf8mb4_general_ci` را انتخاب کن (معمولاً پیش‌فرض است).  
5. دکمهٔ **Create** (یا «ایجاد») را بزن. تمام!

---

### ۴. وارد کردن اطلاعات اولیه (فایل SQL)

حالا باید جدول‌های خالی داخل پایگاه داده را با اطلاعات اولیه پر کنی.

1. در همان phpMyAdmin، از ستون سمت چپ روی نام پایگاه داده‌ای که ساختی (`natural_disaster_db`) کلیک کن.  
2. از منوی بالا، تب **Import** (وارد کردن) را انتخاب کن.  
3. روی دکمهٔ **Choose File** (انتخاب فایل) کلیک کن و داخل پوشهٔ پروژه، فایلی با پسوند `sql.` را پیدا کن.  
   (مثلاً `database.sql` یا `natural_disaster_db.sql`)  
4. تنظیمات دیگر را دست نزن و دکمهٔ **Go** (برو) را بزن.  
5. چند ثانیه صبر کن. اگر پیام سبز رنگ `Import successfully finished` را دیدی، یعنی جدول‌ها ساخته و پر شده‌اند. 🎉

---

### ۵. چک کردن سیم‌های اتصال

پروژه باید بداند به کدام پایگاه داده وصل شود. ما این مسیر را از قبل تنظیم کرده‌ایم، اما یک بررسی کوچک ضرری ندارد.

1. در پوشهٔ پروژه، فایل زیر را با برنامهٔ Notepad (دفترچه یادداشت) باز کن:
   ```
   config/database.php
   ```
2. ببین آیا این اطلاعات را می‌بینی:
   - **host:** `localhost`  
   - **username:** `root`  
   - **password:** (خالی)  
   - **database:** `natural_disaster_db`  
3. اگر همه چیز همین بود، فایل را ببند. اگر مقداری فرق داشت، اصلاح کن و ذخیره کن.

---

### ۶. اجرای نهایی 🎉

1. مرورگر را باز کن و در نوار آدرس تایپ کن:
   ```
   http://localhost/Natural_Disaster_Controller
   ```
   (اگر نام پوشه را عوض کردی، دقیقاً همان را بنویس. حروف کوچک و بزرگ فرقی ندارد.)  
2. کلید Enter را بزن.  
3. اگر صفحهٔ اصلی سامانه با نقشه و منوها ظاهر شد، **تبریک!** تو یک مدیر بحران حرفه‌ای شدی. 🎊

## 🔑 ورود به بخش مدیریت

برای کنترل کامل سامانه، یک پنل مدیریتی وجود دارد. با این اطلاعات وارد شو:

- **آدرس:** `http://localhost/Natural_Disaster_Controller/admin`  
- **نام کاربری:** `admin`  
- **رمز عبور:** `admin123`

> ⚠️ خواهش می‌کنم بعد از اولین ورود، رمز عبور را از داخل پنل تغییر بده.

## 🗄️ یک نگاه سریع به جدول‌های اطلاعاتی

برای آن‌هایی که کنجکاوند داخل کمد اطلاعات چه خبر است:

| نام جدول           | محتوا                                |
|--------------------|--------------------------------------|
| `users`            | حساب کاربری مدیران و امدادگران       |
| `alerts`           | هشدارهای ثبت‌شده                     |
| `disasters`        | انواع بلایای طبیعی (زلزله، سیل و...) |
| `provinces`        | استان‌ها و مناطق                     |
| `citizen_reports`  | گزارش‌های ارسالی از طرف مردم         |
| `recommendations`  | آموزش‌ها و توصیه‌های ایمنی           |
| `contact_messages` | پیام‌های «تماس با ما»                |
| `risk_history`     | تاریخچه تغییرات سطح خطر              |
| `backup_logs`      | گزارش‌های پشتیبان‌گیری               |

نیازی نیست مستقیم با این جدول‌ها کار کنی—سامانه خودش همه چیز را مدیریت می‌کند.

## ❓ اگر چیزی درست کار نکرد (رفع اشکال)

| مشکل | راه‌حل سریع |
|------|--------------|
| **آیکون WampServer سبز نیست** | اگر نارنجی است: روی آیکون کلیک کن، `Apache → Service → Start` و `MySQL → Service → Start` را بزن. اگر قرمز است: از منوی آیکون `Restart All Services` را انتخاب کن. برنامه‌هایی مثل Skype یا TeamViewer را ببند، شاید پورت ۸۰ را اشغال کرده باشند. |
| **خطای 404 (صفحه پیدا نشد)** | مطمئن شو نام پوشه را درست نوشته‌ای و در مسیر `C:\wamp64\www` قرار دارد. یک بار WampServer را Restart کن. |
| **خطای Access denied for user 'root'** | فایل `config/database.php` را باز کن و رمز عبور را چک کن. اگر برای `root` در MySQL رمز گذاشته‌ای، همان را وارد کن؛ وگرنه رمز را خالی بگذار (`''`). |
| **فایل SQL ایمپورت نمی‌شود** | مطمئن شو پایگاه داده `natural_disaster_db` از قبل ساخته شده. گاهی محدودیت حجم آپلود phpMyAdmin مانع می‌شود. راه ساده: فایل SQL را با Notepad باز کن، همهٔ محتوایش را کپی کن، در phpMyAdmin به تب SQL برو، آن را بچسبان و اجرا کن. |
| **صفحه اصلی قاطی به نظر می‌رسد** | احتمالاً فایل‌های CSS و JavaScript کامل بارگذاری نشده‌اند. دکمه‌های Ctrl+F5 را بزن تا کش مرورگر پاک شود. |
